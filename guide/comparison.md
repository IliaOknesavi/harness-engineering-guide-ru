# Сравнение основных реализаций харнессов

Это фактологическое сравнение известных агентных харнессов по состоянию на середину 2025 года. Область развивается стремительно — конкретные детали могут меняться, но архитектурные паттерны устойчивы.

Сжатие контекста через суммаризацию. | Файловая память: MEMORY.md (долгосрочная), ежедневные логи (memory/YYYY-MM-DD.md), wiki/ для структурированных знаний. Поддержка протокола MCP. | Порождение суб-агентов с push-уведомлением о завершении. Оркестратор делегирует задачи, результаты объявляются автоматически. | ✅ Open source |

## Подробный разбор: ключевые отличия

### Стратегия контекста

Как каждый харнесс решает, что показать модели:

```
Aider          : Explicit — user adds files manually (/add, /drop)
Claude Code    : On-demand — agent reads files as needed
Cursor         : Semantic — embeddings index, auto-retrieves relevant code
OpenClaw/Nexu  : Config-driven — AGENTS.md declares what to load
Codex          : Repo-aware — scans structure, loads relevant files
Cline          : IDE-aware — open tabs + workspace files
```

**Компромисс:** Явный контроль (Aider) даёт точность, но требует усилий пользователя. Семантический поиск (Cursor) автоматичен, но может пропустить нужные или подтянуть неправильные файлы. Подход на основе конфигурации (OpenClaw) предсказуем, но требует первоначальной настройки.

### Архитектура памяти

```python
# Type 1: No memory (Claude Code, Codex, Cursor, Cline)
# Each session starts fresh. Context comes from project files.
context = load_project_files()  # That's it

# Type 2: File-based memory (OpenClaw, Nexu)
# Persistent knowledge across sessions, user-editable.
context = (
    load_project_files()
    + load_memory("MEMORY.md")
    + load_daily_log(today)
    + load_wiki_if_relevant(task)
)

# Type 3: Embedded memory (proprietary platforms)
# Stored in vector DB, retrieved by similarity.
context = (
    load_project_files()
    + vector_search(task, memory_store)
)
```

### Расширяемость инструментов

| Подход | Проекты | Плюсы | Минусы |
|---|---|---|---|
| **Протокол MCP** | OpenClaw, Claude Code, Cline | Стандартный протокол, совместимость | Накладные расходы на настройку сервера |
| **Файлы навыков** | OpenClaw, Nexu | Самодостаточны, легко делиться, маркетплейс | Кастомный формат |
| **Только встроенные** | Aider, Codex | Просто, предсказуемо | Ограниченная расширяемость |
| **Интеграция с IDE** | Cursor, Cline | Богатый контекст редактора | Привязка к IDE |

### Мультиагентные паттерны

Большинство харнессов одноагентны. Поддержка мультиагентности заметно различается:

```python
# OpenClaw/Nexu: First-class sub-agent spawning
subagent = spawn(
    task="Research competitor pricing",
    model="gpt-4o",
    tools=["web_search", "web_fetch"],
)
# Result auto-announces when done — no polling needed

# Claude Code: Indirect multi-agent via exec
result = exec("claude-code --print 'Review this PR'")
# Works but no structured communication

# Others: No native multi-agent
# Workaround: multiple terminal sessions, manual coordination
```

## Как выбрать подходящий харнесс

| Если вам нужно... | Рассмотрите |
|---|---|
| Глубокое редактирование кода с интеграцией в IDE | **Cursor** или **Cline** |
| Кодинг-агент на основе терминала | **Claude Code** или **Aider** |
| Изолированное (sandboxed) выполнение задач | **Codex** |
| Мультиагентная оркестрация | **OpenClaw** или **Nexu** |
| Постоянная память между сессиями | **OpenClaw** или **Nexu** |
| Расширяемая экосистема инструментов | **OpenClaw** (навыки + MCP) |
| Минимум настройки, просто работает | **Aider** |

## Чего таблица не показывает

- **Поддержка моделей** — Большинство харнессов модель-агностичны (OpenClaw, Cline, Aider поддерживают много провайдеров). Codex работает только с OpenAI. Claude Code ориентирован на Anthropic.
- **Стоимость** — Сильно варьируется в зависимости от выбора модели, размера контекста и сложности задачи. Это не свойство харнесса, а паттерн использования.
- **Скорость** — Зависит больше от задержки модели, чем от архитектуры харнесса. Поддержка стриминга есть везде.
- **Сообщество** — У open-source-проектов (Aider, Claude Code, Cline, OpenClaw) активные сообщества. Размер сообщества не равен качеству.

## Типичные ошибки

- **Выбор по хайпу** — Берите харнесс, который подходит под ваш рабочий процесс, а не тот, у которого больше всего звёзд на GitHub.
- **Игнорирование памяти** — Если ваши задачи растягиваются на несколько сессий, харнесс без постоянной памяти означает, что контекст придётся объяснять заново каждый раз.
- **Допущение, что «open source» значит «бесплатно»** — Сам харнесс бесплатен; вызовы API модели — нет. Закладывайте бюджет на расходы на API.
